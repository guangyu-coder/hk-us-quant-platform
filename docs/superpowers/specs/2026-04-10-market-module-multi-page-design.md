# 市场模块三分页重构设计

日期：2026-04-10

## 背景

当前市场模块把榜单、股票曲线和订单簿放在同一个页面中，带来了几个问题：

1. 信息密度过高，不同任务互相干扰
2. 榜单区、曲线区、订单簿区同时争抢视觉重点
3. 想专注看榜单时，图表和订单簿会让页面显得拥挤
4. 想单独查看某只股票曲线或订单簿时，没有清晰的独立页面心智

这次重构目标不是继续在现有市场页上修补，而是把市场模块拆成职责明确的三个页面。

## 目标

本次设计要实现：

1. `/market` 直接作为榜单页
2. `/market/chart` 作为股票曲线页
3. `/market/orderbook` 作为订单簿页
4. 三个页面之间有清晰的模块内导航
5. 股票曲线页和订单簿页都支持从榜单页点击进入，也支持通过 `?symbol=` 直达

## 非目标

本次不做：

- 新增复杂技术指标体系
- 多股票对比图
- 高级订单簿分析
- 新的数据源切换
- 收藏、自选股、分享链接优化
- 多页面联动缓存优化

重点只放在“信息架构重组 + 页面职责清晰”。

## 方案选择

### 方案 A：市场模块三页体系

推荐采用。

结构如下：

- `/market`：榜单页
- `/market/chart`：股票曲线页
- `/market/orderbook`：订单簿页

优点：

- 页面职责最清楚
- 符合“榜单页作为主入口”的目标
- 曲线和订单簿都能变成真正独立页面
- 未来继续扩展模块内能力时，结构更稳

缺点：

- 需要重组当前市场页的代码与组件边界

### 方案 B：保留总览页 + 子页

优点：

- 对现有用户更温和

缺点：

- 和“榜单页作为主入口”的目标不一致
- 总览页会继续成为功能堆叠点

### 方案 C：单页多 tab

优点：

- 改造成本最低

缺点：

- 本质上仍然是单页，没有解决“独立页面”的需求

## 最终设计

### 1. 路由结构

市场模块改为三页：

- `/market`
  - 市场榜单页
  - 作为市场模块的主入口
- `/market/chart`
  - 股票曲线页
  - 展示单只股票的价格信息和曲线
- `/market/orderbook`
  - 订单簿页
  - 展示单只股票的盘口/订单簿信息

### 2. 模块导航

保留现有左侧一级导航中的 `市场`。

进入市场模块后，在页面顶部增加模块内二级导航：

- `榜单`
- `股票曲线`
- `订单簿`

导航要求：

- 当前页高亮
- 三个页面布局与标题风格保持一致
- 支持从模块内任一页面切换到其他两个页面

### 3. 榜单页设计

`/market` 作为新的市场主页面。

保留并强化现有榜单能力：

- `美股 / 港股` tab
- `全部 / 涨幅榜 / 跌幅榜`
- 涨跌幅区间筛选
- 搜索

页面职责：

- 专注浏览市场股票列表
- 不再在同页展示大图表和订单簿组件

交互：

- 股票行点击默认进入股票曲线页
- 每行同时提供次级操作：
  - `看曲线`
  - `看订单簿`

视觉方向：

- 更偏行情终端式高信息密度列表
- 强调列对齐、数字对齐、涨跌色彩清晰

### 4. 股票曲线页设计

`/market/chart` 专注一只股票的走势分析。

展示内容：

- 股票代码
- 股票名称
- 最新价
- 涨跌额
- 涨跌幅
- 交易所/币种
- 时间周期切换
- 曲线图区域

进入方式：

- 从榜单页点击股票或 `看曲线`
- 直接访问 `/market/chart?symbol=AAPL`

无 symbol 状态：

- 页面显示空状态提示
- 引导用户从榜单页选择股票，或直接在地址参数中带入 `symbol`

### 5. 订单簿页设计

`/market/orderbook` 专注单只股票盘口信息。

展示内容：

- 股票代码
- 股票名称
- 最新价摘要
- 订单簿/盘口组件

进入方式：

- 从榜单页点击 `看订单簿`
- 直接访问 `/market/orderbook?symbol=0700.HK`

无 symbol 状态：

- 与曲线页一致，显示引导空状态

### 6. 参数与跳转规则

股票曲线页和订单簿页都支持 query 参数：

- `symbol`

示例：

- `/market/chart?symbol=AAPL`
- `/market/orderbook?symbol=0700.HK`

跳转规则：

- 榜单页点股票名：默认去 `/market/chart?symbol=...`
- 榜单页点 `看订单簿`：去 `/market/orderbook?symbol=...`
- 模块内二级导航切换时，如果当前已经选中了 `symbol`，则尽量带上该 `symbol`

### 7. 代码结构重组

当前 `frontend/src/app/market/page.tsx` 中混合了：

- 榜单列表
- 图表区域
- 订单簿组件
- 搜索逻辑
- 市场筛选逻辑

重构后应拆分为更清晰的模块边界：

- 榜单页负责市场列表与筛选
- 曲线页负责股票图表逻辑
- 订单簿页负责订单簿展示
- 共享逻辑抽到市场模块内部 helper / 子组件

复用策略：

- 榜单相关 helper 继续复用当前 `market-page-helpers`
- 股票曲线页复用当前图表数据获取与时间周期逻辑
- 订单簿页复用当前 `MarketDataWidget`
- 市场模块二级导航抽成共享组件

### 8. 兼容与行为变化

行为变化：

- 原 `/market` 不再展示大图表和订单簿
- 原本依赖市场页同屏查看曲线与订单簿的方式，改为单独进入子页面

兼容策略：

- 保持 `/market` 路径不变
- 新增 `/market/chart` 与 `/market/orderbook`
- 榜单页保留当前已有的市场切换、榜单、筛选、搜索能力

### 9. 测试设计

#### 前端路由与页面测试

需要补充：

- `/market` 渲染榜单页
- `/market/chart?symbol=AAPL` 渲染曲线页
- `/market/orderbook?symbol=0700.HK` 渲染订单簿页
- 无 `symbol` 时曲线页和订单簿页显示空状态

#### 交互 smoke

需要覆盖：

- 榜单页切换市场与榜单模式
- 榜单页区间筛选仍然有效
- 从榜单页点击进入曲线页
- 从榜单页点击进入订单簿页
- 在带 `symbol` 的情况下切换二级导航能保留当前股票

#### 回归验证

需要确认：

- 搜索仍按当前市场过滤
- 榜单页数据和现有市场列表接口仍一致
- 曲线页图表数据仍能刷新
- 订单簿页仍能展示当前 symbol 的订单簿组件

## 验收标准

满足以下条件即视为完成：

1. `/market` 成为榜单页主入口
2. `/market/chart` 和 `/market/orderbook` 可独立访问
3. 三个页面都有统一的模块内二级导航
4. 榜单页点击股票可以进入曲线页
5. 榜单页点击操作可以进入订单簿页
6. 曲线页和订单簿页支持 `?symbol=` 直达
7. 无 `symbol` 时有明确空状态引导
8. 原有市场筛选与搜索能力在榜单页继续可用

## 风险与后续

本次重构风险主要有两个：

1. 市场页单文件当前职责较多，拆分时容易把共享状态和逻辑切散
2. 模块内导航与 query 参数保留处理不当时，页面跳转体验会变差

后续可以继续做：

1. 榜单页表格化排版进一步优化
2. 曲线页增加更多分析指标
3. 订单簿页增强盘口深度与成交摘要
