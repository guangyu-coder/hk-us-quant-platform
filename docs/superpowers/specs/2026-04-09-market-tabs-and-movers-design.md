# 市场行情模块改造设计

日期：2026-04-09

## 背景

当前市场行情页存在几个明显问题：

- 港股和美股混在同一个热门股票列表里，市场边界不清楚
- 页面只支持搜索和单标的图表，不支持按当日涨跌幅查看市场机会
- 搜索结果没有明确跟随当前市场上下文
- 当前默认股票池偏小，不能满足“尽可能完整地查看美股/港股行情”的目标

本次改造的目标，是把市场页升级成一个更适合日常扫盘的双市场行情面板。

## 目标

围绕以下体验完成改造：

1. 用户进入市场页后，可以明确切换 `美股` 和 `港股`
2. 每个市场都能查看 `全部 / 涨幅榜 / 跌幅榜`
3. 搜索只在当前市场上下文内进行，避免港股和美股结果混杂
4. 列表优先展示尽可能完整的股票池，而不是只显示少量热门股票
5. 保留当前单标的详情、图表和实时刷新能力

## 非目标

本次不包含以下内容：

- 自定义涨跌幅阈值筛选
- 复杂分页、无限滚动或虚拟列表优化
- 多市场同时并排展示
- 行业、主题、成交额等更复杂的筛选器
- 新的数据供应商接入

## 方案选择

本次采用：

`顶部市场 Tab + 榜单模式`

页面结构：

- 顶部一级 tab：`美股`、`港股`
- tab 下二级模式切换：`全部`、`涨幅榜`、`跌幅榜`
- 搜索框保留，但搜索结果只展示当前市场

选择该方案的原因：

- 比上下双分区更聚焦，单屏信息密度更适中
- 比单表格市场筛选更直观，用户心智更明确
- 最符合“先扫盘，再点进单标的”的使用路径

## 用户体验设计

### 页面结构

市场页从上到下分为 4 个区域：

1. 顶部页头
- 标题：`市场行情`
- 简短说明：当前市场、榜单模式、数据更新时间

2. 市场与榜单控制区
- 一级 tab：`美股`、`港股`
- 二级切换：`全部`、`涨幅榜`、`跌幅榜`
- 搜索框：输入代码或名称
- 刷新按钮：刷新当前市场列表与当前选中标的详情

3. 股票列表区
- 展示当前市场下的股票列表
- 默认按当前榜单模式排序
- 点击某只股票后，右侧或下方详情区更新

4. 单标的详情区
- 复用当前页面已有的单标的图表和实时行情摘要
- 当前选中股票变化时，同步刷新详情区

### 列表字段

股票列表最少展示以下字段：

- 股票代码
- 股票名称
- 交易所
- 最新价
- 涨跌额
- 涨跌幅
- 币种

排序规则：

- `全部`：按代码或默认数据源顺序展示
- `涨幅榜`：按 `changePercent` 从高到低排序
- `跌幅榜`：按 `changePercent` 从低到高排序

### 空状态和降级状态

- 当前市场无数据时：提示“当前市场暂无可展示行情”
- 搜索无结果时：提示“当前市场下未找到匹配标的”
- 数据降级时：继续沿用现有 `meta.status / meta.source / meta.message` 提示机制

## 数据设计

### 市场分类规则

前后端统一使用一套市场归类逻辑：

港股：

- `symbol` 以 `.HK` 结尾
- 或 `exchange` 包含 `HK`、`HKEX`、`HONG KONG`
- 或 `country` 包含 `HONG KONG`

美股：

- 不属于港股归类的股票
- 优先包含 `NASDAQ`、`NYSE`、`AMEX`、`ARCA` 等美股交易所

如果数据源字段不完整：

- 先按 `.HK` 判断港股
- 剩余默认归到美股

### 股票池来源

本次不新增数据提供方，优先复用现有后端：

- `/api/v1/market-data/list`
- `/api/v1/market-data/batch`
- `/api/v1/market-data/search`

后端新增或增强：

1. `market-data/list` 支持按市场返回股票列表
2. 返回结果里尽可能包含：
- symbol
- name / instrument_name
- exchange
- country
- instrument_type

前端拿到股票池后，再批量请求当前市场下股票的最新 quote，并生成榜单。

### 当日涨跌幅数据

涨跌榜基于实时行情返回的以下字段：

- `change`
- `change_percent`

如果个别股票缺少涨跌幅：

- 在 `全部` 模式中允许展示，但排序时放到后面
- 在 `涨幅榜 / 跌幅榜` 中默认过滤掉无涨跌幅数据的股票

## 前端设计

### 状态拆分

市场页新增以下前端状态：

- `selectedMarket`: `US | HK`
- `selectedBoardMode`: `all | gainers | losers`
- `marketSearchQuery`
- `visibleStocks`
- `selectedSymbol`

派生数据：

- 当前市场股票池
- 当前市场搜索结果
- 当前榜单模式排序结果

### 组件职责

在不大幅重写现有页面的前提下，建议逐步拆出这些职责：

- 市场控制区：负责 tab、榜单模式、搜索、刷新
- 股票列表区：负责股票表格和排序后的展示
- 单标的详情区：继续负责图表和实时行情

本次允许先留在同一页文件中实现，但应将市场归类、榜单排序、搜索过滤提取为 helper，避免 `page.tsx` 继续膨胀。

### 搜索行为

搜索规则：

- 搜索输入后，先调用现有 `searchSymbols`
- 再按当前市场做二次过滤
- 用户选中某只股票后：
  - 切换当前 `selectedSymbol`
  - 加载该股票详情和图表

## 后端设计

### 接口改造

优先增强现有接口，而不是新开太多接口。

推荐改造：

1. `GET /api/v1/market-data/list`
- 新增可选参数：`market`
- 支持值：`US`、`HK`
- 返回该市场下尽可能完整的股票列表

2. `POST /api/v1/market-data/batch`
- 保持现状
- 由前端将当前市场股票代码批量传入，拿 quote 数据后做榜单

3. `GET /api/v1/market-data/search`
- 接口保持现状
- 前端按当前市场做过滤
- 如果后端实现成本很低，也可以附加支持 `market` 参数，但不是本次强制要求

### 错误处理

接口失败时：

- 市场股票池失败：页面列表区展示错误提示，不影响已选股票详情区
- 单标的详情失败：沿用现有实时行情降级和提示
- 批量 quote 失败：列表区保留基础股票信息，并提示“当前榜单数据刷新失败”

## 测试策略

### 前端测试

至少覆盖：

1. 市场 tab 切换
- 默认美股
- 切换港股后列表内容变化

2. 榜单模式切换
- `涨幅榜` 按涨幅降序
- `跌幅榜` 按跌幅升序

3. 搜索按市场过滤
- 美股 tab 下搜索不应混入港股结果
- 港股 tab 下搜索不应混入美股结果

4. 选股行为
- 点击列表项后详情区切换到对应股票

### 后端测试

至少覆盖：

1. 市场归类逻辑
- `.HK` 正确识别为港股
- `HKEX / Hong Kong` 正确识别为港股
- 其他默认归入美股

2. `market-data/list` 的市场参数解析
- `US`
- `HK`
- 缺省参数回退逻辑

## 验收标准

完成后应满足：

1. 市场页顶部可切换 `美股 / 港股`
2. 当前市场下可切换 `全部 / 涨幅榜 / 跌幅榜`
3. 榜单按当日涨跌幅正确排序
4. 搜索结果受当前市场约束
5. 点击列表中的股票后，详情区仍可正常展示图表和实时行情
6. 前端类型检查、lint、市场页 smoke 或针对性测试通过

## 风险与边界

### 风险 1：完整股票池规模较大

影响：

- 批量获取 quote 的时间可能变长
- 首屏加载可能比当前热门股票模式更慢

处理：

- 先使用现有接口能力完成功能正确性
- 如果列表过大导致体验明显下降，后续再补缓存、分页或市场内分段加载

### 风险 2：不同数据源市场字段不稳定

影响：

- 个别股票可能被归错市场

处理：

- 优先使用 `.HK` 和常见交易所关键字作为强规则
- 将剩余股票默认归入美股，避免港股漏判造成主要功能失效

### 风险 3：涨跌幅字段不完整

影响：

- 某些股票无法参与涨跌榜排序

处理：

- `全部` 模式继续展示
- `涨幅榜 / 跌幅榜` 自动过滤无涨跌幅数据的股票
